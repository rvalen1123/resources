# Curated MCP servers.
#
# Format (one per line, pipes as delimiters):
#   <short-name> | <one-line description> | <install command>
#
# Lines starting with `#` are comments. Blank lines are ignored.
# `[env: VAR]` markers call out required environment variables.
#
# NOTE: bootstrap/install-mcp.sh only LOGS this file — it does NOT auto-register
# any of these servers. Pick the ones you want and run the install command
# yourself, substituting env vars where needed.
#
# All entries below are publicly available, widely useful servers. Anything
# private/internal belongs in your own notes, not here.
#
# Discovery: the official registry at https://registry.modelcontextprotocol.io/
# is the broader catalog. This file is an opinionated short-list; the registry
# has hundreds more.

# ─── Reference servers (first-party @modelcontextprotocol) ───

filesystem | Read/write files under allowed roots. | claude mcp add filesystem -- npx -y @modelcontextprotocol/server-filesystem /path/to/allowed/root
fetch | Fetch a URL and return cleaned text/markdown. | claude mcp add fetch -- npx -y @modelcontextprotocol/server-fetch
git | Read/search a local git repo: log, diff, blame, show. | claude mcp add git -- npx -y @modelcontextprotocol/server-git --repository /path/to/repo
memory | Persistent knowledge-graph memory across sessions. | claude mcp add memory -- npx -y @modelcontextprotocol/server-memory
sequential-thinking | Structured step-by-step reasoning scratchpad. | claude mcp add sequential-thinking -- npx -y @modelcontextprotocol/server-sequentialthinking
time | Timezone-aware current time and conversions. | claude mcp add time -- npx -y @modelcontextprotocol/server-time
everything | Reference/test server demonstrating prompts, resources, and tools (educational). | claude mcp add everything -- npx -y @modelcontextprotocol/server-everything

# ─── Notes on picking servers ───
#
# 1. Start with `filesystem`, `fetch`, `git`, `memory`, `github` — they cover
#    80% of actual coding workflows.
# 2. Add domain servers (sentry/linear/notion) only when you're actively working
#    in that tool for a sustained stretch. Every active server inflates tool-def
#    size in every prompt and cuts into the prompt cache.
# 3. `memory` is interesting but double-edged — it writes state you may forget
#    about across sessions. Review what it's retained every few weeks.
# 4. For servers requiring tokens, store them in your shell's secret manager
#    (not in `.claude/settings.json`). The bootstrap installer explicitly
#    refuses to install templates containing hardcoded secrets.
